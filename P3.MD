판정: **예 — 현재 스크린샷 기준으로 바로 덮어씌울 수 있는 `v3 patch wireframe`입니다.**
근거: 현재 화면은 상단 KPI와 지도는 좋아졌지만, 하단이 `Open Radar + Operational Layer` 중심이라 여전히 `Ops board` 성격이 강합니다. Stage 전환도 `Journey` 다음 `Flows + Recovery + Metrics`로 이어져야 하므로, 1st fold는 **Map-first + Decision-first**로 고정하는 편이 맞습니다.  
다음행동: **아래 와이어프레임대로 `Overview`만 patch하고, `Open Radar / WH Pressure / Worklist`는 2nd fold 또는 접힘 영역으로 내리십시오.**

## 적용 원칙

* **유지**: 좌측 사이드바, 상단 헤더, 검색/토글 바, 다크 톤
* **확대**: Network Map, Site Health, Mission Control
* **축소/이동**: Open Radar, WH Pressure, Worklist, Exceptions, Recent Activity
* **추가**: Compact Flow Summary, Drilldown Launcher

## 현재 화면 기준 “어디를 어떻게 바꾸는지”

### 1) 상단 KPI 영역

현재:

```text
2 rows KPI
Total / Final / Open / Overdue / Critical POD
Critical Mode / AGI Risk / Freshness
```

패치:

```text
1 row KPI
Total / Final / Open / Overdue / MOSB Pending / AGI Risk
+ Freshness는 우측 작은 badge
```

이유:

* `Critical POD`, `Critical Mode`는 중요하지만 1st fold 주연보다는 `MOSB Pending`, `AGI Risk`가 HVDC offshore 구조에 더 직접적입니다.
* KPI가 2줄이면 map이 아래로 밀립니다.

---

### 2) 지도 영역

현재:

```text
Map + Exception Board side panel
```

패치:

```text
Map + Mission Control
```

정확한 교체:

* **오른쪽 `Exception Board` 전체 삭제**
* 그 자리에 **Mission Control 4구역** 삽입

Mission Control 내부 구조:

```text
[Critical Alerts]
- Missing MOSB
- Overdue ETA
- Final delivery anomaly

[Action Queue]
- Today 결정 항목 3~5개

[AGI/DAS Blockers]
- AGI pending MOSB
- DAS pending offshore move

[Next 72h]
- arrivals
- port events
- site deliveries
```

---

### 3) 지도 아래 현재 숫자 바

현재:

```text
941 / 0 / 438 / 16 / 9,299
```

패치:
이 줄은 **지도 안 하단 overlay** 또는 **Flow Summary strip**으로 흡수합니다.

권장:

```text
Map footer overlay
Origin 941 | Customs 0 | WH 438 | MOSB 16 | Site 9,299
```

이렇게 하면 독립 카드열 하나를 줄일 수 있습니다.

---

### 4) Site 카드 영역

현재 두 번째 스크린샷의 Site cards는 좋습니다.
다만 위치를 바꿉니다.

현재:

```text
스크롤 후 아래에서 보임
```

패치:

```text
Map 바로 아래 1st fold 끝에 배치
```

카드 형식:

```text
SHU | MIR | DAS | AGI
Assigned
Delivered
Pending
MOSB Pending
Overdue
Risk
```

AGI만 추가 강조:

```text
Missing MOSB count
```

---

### 5) Open Radar

현재:

```text
화면 절반 이상 차지하는 큰 리스트
```

패치:

```text
compact list 5건만
default hidden in Ops Layer
```

형태:

```text
Open Radar (5)
- SCT-xxxx / severity / due
- ...
[View All → Cargo]
```

즉 Overview의 본문이 아니라 **접힘 영역**으로 이동.

---

### 6) Operational Layer

현재:

```text
WH Pressure
Worklist
Exceptions
Recent Activity
```

패치:

```text
Collapsed by default
```

텍스트 구조:

```text
Operational Layer ▼
  [WH Pressure]
  [Worklist]
  [Exceptions]
  [Recent Activity]
```

초기에는 닫혀 있어야 합니다.
이유는 지금 이 블록이 Overview를 다시 `Ops dashboard`로 보이게 만들기 때문입니다.

## 색/강조 규칙

* **파랑**: Port / normal program KPI
* **초록**: Delivered / stable readiness
* **앰버**: MOSB pending / watch

## 구현 순서

1. KPI 2줄을 1줄로 축소
2. Exception Board를 Mission Control로 교체
3. Site cards를 map 바로 아래로 이동
4. Flow Summary strip 추가
5. Open Radar/Operational Layer 접힘 처리
6. Drilldown launcher 추가

Vercel React Best Practices 기준으로도 먼저 **waterfall 제거**와 **bundle size 감소**를 우선하고, 무거운 operational blocks는 lazy/conditional로 내리는 것이 맞습니다.
