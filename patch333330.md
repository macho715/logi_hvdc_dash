판정: **조건부 — `OverviewMap.tsx`만으로 UAE 네트워크맵의 80%는 고칠 수 있지만, `UAE Ops 기본값`까지 맞추려면 `OverviewPageClient.tsx` 1줄 수정이 필요합니다.**
근거: 현재 `OverviewMap` prop은 `onNavigateIntent`만 받고 있어 track 개념이 없고, 레이어도 `origin arcs + geofence + heatmap + eta wedge + location + trips + status rings + poi`를 한 번에 올립니다. 또 trip tooltip은 아직 `Flow ${tooltip.flowCode}`를 노출합니다. 한편 `dashMode` 기본값은 `program`이고 `OverviewMap`에 전달되지 않습니다.   
다음행동: **`OverviewMap.tsx`는 `track-aware semantic map`으로 바꾸고, 호출부에서만 `track={dashMode === 'ops' ? 'uae-ops' : 'global'}`를 넘기십시오.**

# `OverviewMap.tsx` 실제 패치 방향

## 1) 먼저 결론

이 파일의 핵심 문제는 3개입니다.

1. **track prop이 없다**
2. **UAE Ops 전용 semantic layer가 없다**
3. **tooltip이 아직 Flow Code 언어를 쓴다**

그래서 패치 목표는 아래 한 줄입니다.

```ts
OverviewMap.tsx
= global / uae-ops 분기
+ customs synthetic node
+ UAE semantic edges
+ subdued trips
+ track별 legend/footer/tooltip
```

## 3) tooltip 타입부터 고쳐야 함

현재 trip tooltip 구조는 `vendor / flowCode / etaUnix`만 들어가고, 화면에도 `Flow ${tooltip.flowCode}`가 직접 노출됩니다.  
UAE Ops에서는 이게 틀립니다.

### 교체 타입

```ts
type TooltipInfo =
  | { kind: 'location'; x: number; y: number; object: Location & { status?: LocationStatus } }
  | { kind: 'poi'; x: number; y: number; text: string }
  | { kind: 'arc'; x: number; y: number; country: string; count: number }
  | {
      kind: 'trip'
      x: number
      y: number
      vendor: string | null
      shipmentId?: string | null
      etaUnix: number
      routeLabel?: string
      nextMilestone?: string
      stageLabel?: string
    }
  | {
      kind: 'customs'
      x: number
      y: number
      name: string
      summary?: string
    }
```

### trip hover도 교체

현재:

```ts
const obj = info.object as { vendor: string | null; flowCode: number | null; etaUnix: number }
setTooltip({ kind: "trip", x: info.x, y: info.y, vendor: obj.vendor ?? null, flowCode: obj.flowCode ?? null, etaUnix: obj.etaUnix })
```

교체:

```ts
const obj = info.object as {
  vendor?: string | null
  shipmentId?: string | null
  etaUnix: number
  routeLabel?: string
  nextMilestone?: string
  stageLabel?: string
}

setTooltip({
  kind: 'trip',
  x: info.x,
  y: info.y,
  vendor: obj.vendor ?? null,
  shipmentId: obj.shipmentId ?? null,
  etaUnix: obj.etaUnix,
  routeLabel: obj.routeLabel ?? 'In transit',
  nextMilestone: obj.nextMilestone,
  stageLabel: obj.stageLabel,

```

## 4) navigation intent에 customs 추가

현재 `buildLocationIntent()`는 `site / port / mosb / warehouse`만 처리하고, `buildPoiIntent()`도 `site / port / warehouse / mosb`만 처리합니다. customs intent가 없습니다. 

### 추가 규칙

```ts
function buildCustomsIntent(customsId: string): NavigationIntent {
  return {
    destinationId: 'map-customs',
    page: 'chain',
    params: { focus: 'customs', customs: customsId },
  }
}
```

### 안전장치

```ts
[ASSUMPTION] /chain 페이지가 focus=customs를 아직 처리하지 않으면
1차 패치에서는 /chain?focus=port fallback 사용
```

## 5) UAE 전용 semantic graph를 이 파일에서 memo로 만들어야 함

현재 레이어는 모두 원본 데이터 바로 투입형입니다. `OverviewMap` 내부에서 `tripsData`, `POI_LOCATIONS`, 필요 시 `locations`를 조합해 UAE semantic graph를 만들어야 합니다. 현재는 그런 중간 변환 계층이 없습니다.

### 권장 타입

```ts
type UaeNodeKind = 'port' | 'customs' | 'warehouse' | 'mosb' | 'site'
type UaeEdgeKind =
  | 'port_to_customs'
  | 'customs_to_wh'
  | 'customs_to_site'
  | 'customs_to_mosb'
  | 'wh_to_site'
  | 'wh_to_mosb'
  | 'mosb_to_site'

interface UaeNode {
  id: string
  name: string
  kind: UaeNodeKind
  coordinates: [number, number]
  siteCode?: 'SHU' | 'MIR' | 'DAS' | 'AGI'
}

interface UaeEdge {
  id: string
  source: [number, number]
  target: [number, number]
  kind: UaeEdgeKind
  shipmentCount?: number
  highlighted?: boolean
}
```

### 이 파일 안에서 최소 memo

```ts
const customsNodes = useMemo(() => buildCustomsAnchors(), [])
const uaeGraph = useMemo(() => {
  return buildUaeOpsGraph({
    trips: tripsData,
    pois: POI_LOCATIONS,
    highlightedShipmentId,
  })
}, [tripsData, highlightedShipmentId])
```

## 7) 레이어 스택을 track 기준으로 분기해야 함

현재 이 파일은 항상 같은 layers 배열을 씁니다. `createOriginArcLayer`, `createGeofenceLayer`, `createHeatmapLayer`, `createEtaWedgeLayer`, `createLocationLayer`, `createTripsLayer`, `createStatusRingLayer`, `poiLayers`가 같이 올라갑니다.
이게 UAE 맵을 흐리게 만드는 가장 큰 원인입니다.

## Global 모드

```ts
const globalLayers = [
  createOriginArcLayer(originsData, showOriginArcs),
  createGeofenceLayer(locations, showGeofence),
  createHeatmapLayer(filteredEvents, {
    getWeight: geofenceWeight,
    radiusPixels: heatmapRadiusPixels,
    visible: showHeatmapLayer,
  }),
  createEtaWedgeLayer(locations, showEtaWedge),
  createLocationLayer(locations, statusByLocationId, handleHover, handleLocationClick, {
    visible: showStatusLayer,
  }),
  createTripsLayer(tripsData, animTime, layerTrips, highlightedShipmentId),
  createStatusRingLayer(POI_LOCATIONS, showPoiLayer, stageKey),
  ...poiLayers,
]
```

## UAE Ops 모드

```ts
const uaeLayers = [
  createGeofenceLayer(locations, showGeofence),
  createUaeOpsEdgeLayer(uaeGraph.edges, {
    visible: true,
    onHover: handleHover,
  }),
  createCustomsNodeLayer(customsNodes, {
    visible: true,
    onHover: handleHover,
    onClick: (node) => navigate(buildCustomsIntent(node.id)),
  }),
  createLocationLayer(locations, statusByLocationId, handleHover, handleLocationClick, {
    visible: showStatusLayer,
    opacity: 0.18,
  }),
  createTripsLayer(tripsData, animTime, layerTrips, highlightedShipmentId, {
    subdued: true,
    useSemanticLabels: true,
  }),
  createStatusRingLayer(POI_LOCATIONS, showPoiLayer, stageKey),
  ...poiLayers,
]
```

### 실제 선택

```ts
const layers =
  track === 'uae-ops'
    ? uaeLayers.filter(Boolean)
    : globalLayers.filter(Boolean)
```

## 8) UAE 모드에서 반드시 꺼야 할 것

현재 `showOriginArcs`는 zoom + store 기반으로 켜집니다. UAE 모드에서는 이걸 강제로 꺼야 합니다. 

### 규칙

```ts
const isUaeOps = track === 'uae-ops'
const showOriginArcs = !isUaeOps && layerOriginArcs && zoom <= MAP_LAYER_ZOOM_THRESHOLDS.originArcMax
const showHeatmapLayer = !isUaeOps && showHeatmap && zoom < MAP_LAYER_ZOOM_THRESHOLDS.heatmapMax
```

### 이유

UAE Ops에서는 아래가 금지입니다.

* origin arc 유지
* heatmap 기본 강조
* 모든 선이 UAE 중앙 허브처럼 읽히는 상태

## 15) 꼭 같이 바꿔야 하는 호출부 1줄

`OverviewMap.tsx`만으로는 기본 모드를 바꿀 수 없습니다. 현재 `dashMode` 기본값은 `program`이고, `OverviewMap` 호출 시 아무 prop도 전달하지 않습니다.

### 호출부 최소 수정

```tsx
<OverviewMap
  track={dashMode === 'ops' ? 'uae-ops' : 'global'}
  onNavigateIntent={handleNavigate}
/>
```

### 기본값도 변경

```ts
const [dashMode, setDashMode] = useState<'program' | 'ops'>('ops')
```

이 두 줄을 안 바꾸면, 당신이 요구한 **“Overview 기본값은 UAE Ops”**는 성립하지 않습니다.
